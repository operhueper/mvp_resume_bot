# Спецификация: Расширенное интервью и умный парсинг для MVP Resume Bot

## Executive Summary
Бот-помощник для создания резюме обновляется: внедряется умный парсинг PDF/DOCX файлов, проактивная коучинговая помощь в выборе профессии и строгий алгоритм защиты от галлюцинаций ИИ. Цель — создать инструмент, который глубоко погружается в опыт пользователя, помогает вытащить поводы для гордости даже при нулевом опыте (школа, университет) и выдает резюме, которым соискатель будет реально гордиться.

## Problem Statement
Текущая версия бота прямолинейна и не гибка.
- Пользователь вынужден вручную вводить всё, даже если у него уже есть резюме в PDF/DOCX.
- Бот "фантазирует" (галлюцинирует), когда его просят о помощи.
- Бот задает мало уточняющих вопросов по навыкам и не анализирует предыдущие блоки, из-за чего финальное резюме получается сухим. 
- Нет пути для новичков (без опыта работы) и людей с низкой самооценкой (не знают, кем хотят работать).

## Success Criteria
- **Zero Hallucinations:** ИИ оперирует только фактами, предоставленными пользователем.
- **AI-Parser Integration:** Бот успешно извлекает опыт, навыки и образование из PDF/DOCX и пропускает соответствующие шаги опроса.
- **Deep Skills Extraction:** Бот задает до 2-х точечных уточняющих вопросов по каждому навыку/месту работы.
- **Pride & ATS-Ready:** Финальное резюме выглядит профессионально, содержит метрики и описывает сильные стороны кандидата (проверено ботом на соответствие требованиям ATS).

## User Personas
- **Соискатель с опытом:** Имеет старое резюме, хочет обновить и улучшить его для современных ATS систем.
- **"Потерянный" профессионал:** Обладает навыками, но не уверен, как они называются на рынке труда, и нуждается в помощи.
- **Студент/Новичок (0 опыта):** Не имеет коммерческого опыта, нуждается в выявлении потенциала через учебу, хобби и стремления.

## User Journey
1. **Точка входа (`/start`):** Бот спрашивает, знает ли пользователь, кем хочет работать.
   - *Путь А:* "Я знаю" -> Ввод должности.
   - *Путь Б:* "Помоги выбрать" -> 2-3 коучинговых вопроса (что нравится, сильные стороны) -> Бот предлагает 3 варианта -> Выбор.
2. **Загрузка файла:** Предложение загрузить PDF/DOCX.
   - Если файл загружен `gpt-4o` парсит его и раскладывает опыт/навыки по внутреннему JSON (Supabase).
3. **Анализ и дополнение (Интервью):**
   - Бот пропускает идеально заполненные блоки.
   - Для пустых/слабых блоков говорит: *"Я прочитал твой опыт в X, но не вижу достижений. Для ATS-систем они очень важны. Давай добавим их."*
   - Если опыта нет (0 мест) -> Переход к блоку "Гордость в университете/школе/хобби", превращая это в раздел "Проекты/Опыт".
4. **Глубокий разбор навыков:**
   - Для заявленных навыков бот задает максимум 2 уточняющих вопроса (например: "Какой конкретно фреймворк?").
   - Если ответов нет, бот мягко спрашивает: *"Уверен ли ты, что данных достаточно?"* и переходит дальше.
5. **Финализация:**
   - Бот предлагает подходящие вакансии по собранному профилю.
   - Предлагает сгенерировать резюме, которым можно гордиться.

## Functional Requirements
### Must Have (P0)
- [ ] **Onboarding Branching:** Переработка `start.py` для выбора профессии или запроса коучинга.
- [ ] **Document Parsing Pipeline:** Извлечение текста из PDF/DOCX и маппинг в схему `rb_users.interview_state`.
- [ ] **Hidden Classification Step:** Использование `gpt-4o-mini` для анализа сообщений (запрос помощи, неуверенность) перед генерацией ответа.
- [ ] **Anti-Hallucination Prompt:** Строгие системные инструкции для недопущения придумывания фактов ИИ.

### Should Have (P1)
- [ ] **ATS Evaluator:** Бот должен комментировать каждый блок с точки зрения ATS-систем (хватает ли ключевых слов, метрик).
- [ ] **Zero-Experience Optimizer:** Извлечение навыков из школьных успехов, вуза, турниров и хобби.

### Nice to Have (P2)
- [ ] **Vacancy Matcher:** Генерация названий конкретных вакансий (например: "Junior Project Manager", "Client Success Manager") в конце опроса на базе собранного JSON.

## Technical Architecture
### Data Model (Supabase)
- Использование существующей `rb_users.interview_state` (JSONB). 
- Необходимо обогатить JSON ключами для хранения отпарсенных данных (`parsed_from_file: boolean`, `ats_score: number`).

### System Components
- **Classifier Node:** Скрытая логика в стейтах интервью, классифицирующая интент пользователя (`answers_question`, `needs_help`, `confused`, `wants_to_skip`).
- **Parser Node:** Модуль на базе `gpt-4o` для распознавания PDF/DOCX и конвертации в JSON формат стейта.

### AI Model Strategy
- `gpt-4o-mini`: Скрытая классификация интентов, коучинговые вопросы при старте, генерация уточняющих вопросов (не более 2х).
- `gpt-4o`: Тяжеловесный парсинг PDF/DOCX в JSON, генерация финального резюме без галлюцинаций.

## Non-Functional Requirements
- **Performance:** Скрытая классификация (gpt-4o-mini) должна работать быстро (< 1-2 сек), чтобы не задерживать диалог.
- **Reliability:** Парсинг документов должен корректно обрабатывать битые файлы или изображения внутри PDF (возвращать понятную ошибку юзеру).

## Out of Scope
- Распознавание не текстовых файлов (чистые картинки JPG/PNG).
- Автоматическая рассылка резюме работодателям.
