# Резюме-бот MVP — Спецификация

## Executive Summary

Telegram-бот для создания качественных резюме через диалоговое интервью с AI. Бот помогает соискателям всех профессий сформулировать свой опыт в формате, понятном HR и оптимизированном для ATS-систем. MVP фокусируется на качественном резюме в текстовом формате.

## Проблема

1. **Люди не умеют писать резюме** — не знают структуру, не умеют формулировать достижения в цифрах
2. **91% крупных компаний используют AI-скрининг** — 75% резюме отклоняются автоматически из-за неправильного формата
3. **Рутина и хаос** — соискатели тратят часы на однотипные действия и теряют отклики

## Критерии успеха

| Метрика | Цель |
|---------|------|
| Activation | >60% дошли до первого черновика |
| Completion | >40% сделали экспорт |
| Time-to-value | <20 минут до первого черновика |
| Satisfaction | >7/10 средняя оценка |

## Целевая аудитория

**Широкая аудитория соискателей:**
- Все профессии: IT, маркетинг, продажи, производство, услуги
- Все уровни: от джуна до топ-менеджера
- Технический уровень: низкий-средний (пользователи Telegram)
- Язык: русский

**Персона:**
> Мария, 28 лет, маркетолог. Хочет сменить работу, но не обновляла резюме 3 года. Не знает как описать достижения, боится что резюме "слабое". Готова потратить 20-30 минут на хороший результат.

## User Journey

### Первый контакт (0-2 мин)
```
Пользователь → /start → Приветствие + объяснение что умеет бот
                      → Вопрос: "Как может называться ваша идеальная позиция?"
                      → Пользователь отвечает свободным текстом
```

### Онбординг (2-5 мин)
```
Бот определяет профессиональную область из ответа
→ Спрашивает базовую информацию:
  - Имя
  - Контакты (телефон, email, город)
  - Текущий статус (работаю/ищу/на паузе)
→ Предлагает загрузить старое резюме (опционально)
  - Если да → AI-парсинг → заполнение профиля
  - Если нет → продолжаем интервью
```

### Интервью (10-25 мин)
```
Этапы интервью (показывается прогресс "Этап 2/6"):

1. Summary / О себе
   - "Расскажите в 2-3 предложениях, чем вы занимаетесь"
   - Бот формирует профессиональное summary

2. Опыт работы (по каждому месту)
   - Компания, должность, даты
   - "Какие были основные обязанности?"
   - "Какой главный результат вы достигли?"
   - Уточнения: "Можете назвать цифры? Например: +30% продаж, 50 клиентов"
   - Подсказки инструментов по профессии

3. Навыки
   - Текстовые подсказки: "Возможно, вы используете: Figma, Photoshop?"
   - Свободный ввод дополнительных навыков

4. Образование
   - ВУЗ, специальность, год
   - Курсы и сертификаты

5. Дополнительно (опционально)
   - Портфолио/GitHub
   - Языки
   - Пожелания по работе

→ После каждого блока: показать черновик → "Всё верно? / Исправить"
```

### Генерация (1-2 мин)
```
Бот анализирует собранные данные
→ Предлагает 2-3 варианта названия позиции на основе знаний о рынке
→ Пользователь выбирает или вводит своё
→ Генерация резюме в текстовом формате
→ Показ результата в чате
→ Предложение сохранить / отредактировать
```

### Редактирование
```
Команды:
- "Сделай короче"
- "Перепиши блок опыта в [компания]"
- "Добавь навык X"
- "Сделай формальнее/проще"

→ Бот применяет изменения → показывает результат
```

### Экспорт
```
/export → Текст резюме в сообщении (копипаст для hh.ru)
        → (будущее) PDF-файл
```

## Критерии качества резюме

### Структура (ATS-оптимизация)
- Одноколоночный макет
- Стандартные секции: Summary, Experience, Skills, Education
- 1-2 страницы максимум (для российского рынка — 1 страница)
- Формат: .docx или .pdf (текст для MVP)

### Буллеты достижений (формула)
```
[Глагол действия] + [что сделал] + [результат с метрикой] + [инструменты]

Примеры:
✓ "Увеличил конверсию лендинга на 40% за счёт A/B-тестирования (Amplitude, Optimizely)"
✓ "Автоматизировал отчётность, сократив время подготовки с 8 часов до 30 минут (Python, Pandas)"
✗ "Занимался маркетингом и рекламой" — слабо, нет результата

Цель: 70%+ буллетов с цифрами
```

### Глаголы действия (по категориям)
```
Управление: руководил, координировал, делегировал, контролировал
Достижения: увеличил, сократил, оптимизировал, внедрил
Создание: разработал, создал, запустил, спроектировал
Анализ: проанализировал, исследовал, выявил, оценил
```

### Проверки качества
- [ ] Орфография и грамматика
- [ ] Консистентность дат (нет пересечений)
- [ ] Нет "воды" без фактов
- [ ] Каждый опыт имеет хотя бы 1 достижение
- [ ] Skills сгруппированы логически

### База подсказок навыков (примеры)

| Профессия | Инструменты |
|-----------|-------------|
| Frontend | JavaScript, TypeScript, React, Vue, CSS, Figma |
| Backend | Python, Node.js, PostgreSQL, Docker, Git |
| Дизайнер | Figma, Photoshop, Illustrator, Sketch |
| Маркетолог | Google Analytics, Яндекс.Метрика, Facebook Ads, CRM |
| Продажи | CRM, 1С, Excel, переговоры, презентации |
| HR | HRM-системы, 1С:ЗУП, Excel, интервьюирование |
| PM | Jira, Confluence, Miro, Agile, Scrum |

## Техническая архитектура

### Стек технологий

**Backend:**
- Python 3.11+ с aiogram (Telegram Bot API)
- FastAPI для webhook'ов и админ-API
- Pydantic для валидации

**AI:**
- Claude API (Anthropic) — основная модель для генерации
- Рекомендация: claude-3-haiku для интервью (быстро, дёшево), claude-3-sonnet для финальной генерации

**База данных:**
- Supabase (PostgreSQL + Auth + Storage)

**Парсинг документов:**
- PyMuPDF или pdfplumber для PDF
- python-docx для DOCX
- AI для структурированного извлечения данных

**Админ-панель:**
- React или Next.js
- Supabase Auth для авторизации админа
- Recharts для графиков воронки

**Хостинг:**
- Зарубежный: Vercel (фронт) + Railway/Render (бэкенд)
- Или: VPS на Hetzner

### Схема данных

```sql
-- Пользователи
users (
  id UUID PRIMARY KEY,
  telegram_id BIGINT UNIQUE,
  telegram_username TEXT,
  created_at TIMESTAMP,
  last_active_at TIMESTAMP,
  current_stage TEXT, -- 'onboarding', 'interview', 'draft', 'exported'
  interview_state JSONB -- текущее состояние интервью
)

-- Профили кандидатов (нормализованные данные)
candidate_profiles (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users,
  full_name TEXT,
  email TEXT,
  phone TEXT,
  city TEXT,
  desired_position TEXT,
  summary TEXT,
  raw_data JSONB, -- всё что собрали
  created_at TIMESTAMP,
  updated_at TIMESTAMP
)

-- Опыт работы
work_experiences (
  id UUID PRIMARY KEY,
  profile_id UUID REFERENCES candidate_profiles,
  company TEXT,
  position TEXT,
  start_date DATE,
  end_date DATE,
  is_current BOOLEAN,
  responsibilities TEXT[],
  achievements JSONB[], -- [{text, metrics, tools}]
  order_index INT
)

-- Резюме (может быть несколько)
resumes (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users,
  profile_id UUID REFERENCES candidate_profiles,
  title TEXT, -- название позиции
  content TEXT, -- сгенерированный текст
  version INT,
  created_at TIMESTAMP
)

-- Навыки
skills (
  id UUID PRIMARY KEY,
  profile_id UUID REFERENCES candidate_profiles,
  name TEXT,
  category TEXT -- 'hard', 'soft', 'tool'
)

-- Образование
education (
  id UUID PRIMARY KEY,
  profile_id UUID REFERENCES candidate_profiles,
  institution TEXT,
  degree TEXT,
  field TEXT,
  year INT
)

-- Аналитика действий
events (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users,
  event_type TEXT,
  metadata JSONB,
  created_at TIMESTAMP
)
```

### Интеграции

| Сервис | Назначение | Приоритет |
|--------|-----------|-----------|
| Telegram Bot API | Основной интерфейс | P0 |
| Claude API | Генерация и парсинг | P0 |

## Безопасность и приватность

### Хранение данных
- Данные хранятся бессрочно до запроса на удаление
- Команда /delete_profile полностью удаляет профиль и все резюме
- PII (телефон, email) хранятся в зашифрованном виде

### Правила этики
- Бот НЕ выдумывает опыт — только структурирует то, что сказал пользователь
- Если данных мало — спрашивает, а не додумывает
- Перед экспортом — напоминание "проверьте факты"

## Диалоговая логика

### Стиль общения
- Профессиональный HR-тон
- Без эмодзи (или минимум)
- Один вопрос за раз
- Чёткие инструкции

### Примеры диалогов

**Начало:**
```
Бот: Привет! Я помогу создать сильное резюме, которое заметят HR и пропустят ATS-системы.

Как может называться позиция, которую вы ищете?
Например: "Frontend-разработчик", "Маркетолог", "Менеджер по продажам"

Пользователь: хочу работать продактом

Бот: Отлично, будем делать резюме на позицию Product Manager.
Давайте начнём с базовой информации.
Как вас зовут?
```

**Уточнение достижений:**
```
Бот: Этап 3/6: Опыт работы

Расскажите о вашей работе в [компания].
Какой главный результат вы там достигли?

Пользователь: ну я там делал разные задачи по продукту

Бот: Понял. А можете вспомнить конкретный результат?
Например:
- Запустили новую фичу, которая увеличила X?
- Сократили время/затраты на что-то?
- Выросла какая-то метрика?

Пользователь: да, мы запустили онбординг и конверсия выросла

Бот: Отлично! А на сколько процентов выросла конверсия?

Пользователь: не помню точно, примерно процентов 20

Бот: Хорошо, запишу: "Запустил новый онбординг, увеличив конверсию на ~20%"
Это верно?
```

**Офф-топик:**
```
Пользователь: а ты можешь найти мне вакансии?

Бот: Пока я специализируюсь только на создании резюме. Поиск вакансий появится в будущих версиях.
Давайте продолжим с резюме — мы на этапе [X].
[следующий вопрос]
```

## Не входит в MVP (Out of Scope)

1. **CRM для соискателя** — Kanban-доска отслеживания откликов
2. **Автоотклики** — автоматическая подача на вакансии
3. **Парсинг TG-каналов** — сбор вакансий из Telegram
4. **Парсинг hh.ru** — актуальные названия позиций
5. **PDF-экспорт** — красивые шаблоны
6. **Голосовые сообщения** — расшифровка через Whisper
7. **Монетизация** — подписки, платежи
8. **Мультиязычность** — английский интерфейс и резюме

## План реализации (1-2 недели)

### Неделя 1
- День 1-2: Настройка проекта, Supabase, базовый бот
- День 3-4: Интервью-логика, сохранение состояния
- День 5-6: AI-генерация резюме, подсказки навыков
- День 7: AI-парсинг документов

### Неделя 2
- День 1-2: Редактирование, несколько резюме
- День 3-4: Админ-панель с воронкой
- День 5: Тестирование, исправление багов
- День 6-7: Деплой, мониторинг
